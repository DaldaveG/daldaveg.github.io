Job Description

As a web developer in the enterprise content services team, my main responsibilities were to create web pages for internal company clients. Assisting with support tickets from clients. This was a good way to reacquaint me with the web management I used last year, and I was up to speed within the first weeks. The tickets ranged anywhere from being requests to update, change, or delete content, to more challenging ones, like finding what is causing a display defect on a page. In both these cases, I used HTML/CSS and got back to the clients. When I had some free time (usually in between different projects) I was given some miscellaneous tasks. I used JavaScript to create a function that would redirect users from a page after a certain amount of time elapsed for security purposes on a site. I was then responsible for handling the updating of a Spring web form for a client. I added and new logic needed for the interaction. I also implemented and added the new content to be sent with the email verification. I then demoed the new form for the client during a meeting where I walked through the process of completing the form while highlighting the key changes. I was also involved in testing and QA work for a software platform upgrade, where I utilized that all components of the platform worked as expected. Before my work term ended, I helped with the migration of an old site to the current WCM being used.
